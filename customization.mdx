## Brand Color and Theme Settings

Update your space's appearance starting with the brand color. Use `#3B82F6` as the default blue accent for buttons, links, and highlights.

<Steps>
  <Step title="Access Settings" icon="settings" title-type="p">
    Navigate to Space Settings > Appearance.
  </Step>

  <Step title="Enter Brand Color" icon="palette" title-type="p">
    Input your hex code, such as `#3B82F6`.
  </Step>

  <Step title="Preview and Save" icon="eye" title-type="p">
    Preview changes live, then save to apply across your documentation.
  </Step>
</Steps>

<Callout kind="tip" collapsed="false">
  Test multiple shades of your brand color to ensure readability on both light and dark themes.
</Callout>

For programmatic configuration, edit your `config.json` file:

<CodeGroup show-lines="true" tabs={[]}>
  ```json
  {
    "theme": {
      "brandColor": "#3B82F6",
      "primaryFont": "Inter",
      "logoUrl": "https://your-site.com/logo.png"
    }
  }
  ```

  ```yaml
  theme:
    brandColor: "#3B82F6"
    primaryFont: Inter
    logoUrl: https://your-site.com/logo.png
  ```
</CodeGroup>

## Layout and Navigation Options

Adapt the layout for better navigation. Choose between sidebar, topbar, or full-width views.

<Tabs>
  <Tab title="Sidebar Layout" icon="sidebar">
    Ideal for deep documentation hierarchies.

    Enable via settings:

    ```
    layout: "sidebar"
    navigation: "nested"
    ```

    This positions menus on the left for quick access.
  </Tab>

  <Tab title="Topbar Layout" icon="menu">
    Suited for shallow content structures.

    ```
    layout: "topbar"
    navigation: "flat"
    ```

    Keeps options compact at the top.
  </Tab>

  <Tab title="Custom Navigation" icon="settings">
    Combine elements for hybrid setups.

    <Expandable title="Advanced Config" default-open="false">
      Use arrays for custom menu items:

      ```javascript
      navigation: [
        { title: "API", href: "/api" },
        { title: "Guides", href: "/guides" }
      ]
      ```
    </Expandable>
  </Tab>
</Tabs>

## Export and Import Functions

Export your entire space as Markdown or JSON for backups or migrations. Import from GitHub repos or ZIP files.

<Steps>
  <Step title="Export Space" icon="download" title-type="p">
    Go to Settings > Export. Select format (MDX or JSON) and download.
  </Step>

  <Step title="Import Content" icon="upload" title-type="p">
    Use Import > Select file or Git URL. Map paths automatically.
  </Step>

  <Step title="Verify Import" icon="check-circle" title-type="p">
    Review imported pages in preview mode before publishing.
  </Step>
</Steps>

## Access Controls

Secure your documentation with role-based permissions. Define viewers, editors, and admins.

| Role   | View | Edit | Admin |
| ------ | ---- | ---- | ----- |
| Viewer | ✅    | ❌    | ❌     |
| Editor | ✅    | ✅    | ❌     |
| Admin  | ✅    | ✅    | ✅     |

<Callout kind="alert" collapsed="false">
  Always enable two-factor authentication for admin roles to protect sensitive content.
</Callout>

Configure via API for automation:

```javascript
// Set user permissions
await fetch('/api/space/permissions', {
  method: 'POST',
  body: JSON.stringify({
    userId: 'user123',
    role: 'editor'
  })
});
```
