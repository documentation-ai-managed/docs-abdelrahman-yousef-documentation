## Who this guide is for

This guide helps first-time users understand how to begin using the Abdelrahman Yousef Documentation platform effectively. It summarizes the onboarding flow, introduces essential concepts, and points you to the most relevant next steps.

If you have not yet reviewed the foundational pages, start with the [Introduction](/introduction) and follow the [Quickstart](/quickstart) to set up your initial documentation space.

## Your first hour

Follow this sequence for a smooth onboarding experience.

<Steps>
  <Step title="Review the Introduction" icon="book-open">
    
    Read the [Introduction](/introduction) to understand the purpose of the platform and the core features available to you.
  
  </Step>

  <Step title="Complete the Quickstart" icon="zap">
    
    Follow the [Quickstart](/quickstart) to create your space, verify your account, and publish your first MDX page.
  
  </Step>

  <Step title="Explore your docs structure" icon="folder">
    
    Open the navigation sidebar and inspect how pages are organized. Use [Managing Docs](/managing-docs) to learn how to add, move, or rename pages.
  
  </Step>

  <Step title="Customize branding" icon="palette">
    
    Adjust primary colors or other visual options following the [Customization](/customization) guide.
  
  </Step>

  <Step title="Browse the API Reference" icon="code">
    
    Familiarize yourself with the interactive API explorer at [openapi.yaml](/openapi.yaml). This is useful when documenting technical workflows.
  
  </Step>
</Steps>

## Quick checklist

Use this as a short reference to confirm you have completed the essentials.

- Read the introduction page.
- Completed account setup and initial project creation.
- Created and published at least one MDX page.
- Adjusted your navigation and added a few sample sections.
- Applied a brand color or theme adjustments.
- Viewed the API reference and changelog.
